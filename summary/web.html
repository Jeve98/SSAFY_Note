Web

World Wide Web, 통칭 Web은 인터넷 상에서 각종 멀티미디어 정보를
하이퍼텍스트 방식으로 연결하여 제공하는 서비스를 말합니다.
Web은 HTML, CSS, Javascript 등 3가지 구성요소로 이루어집니다.

HTML (HyperText Markup Language)

- HyperText : 다른 웹 페이지로 연결/참조하는 링크로 비선형적, 상호연결적, 사용자 주도적 탐색을 지원합니다.
- Markup Language : 태그 등을 이용하여 문서나 데이터의 구조를 명시하는 언어로 html에서는 내부의 내용이 없을 경우,
닫는 태그가 없는 경우도 있습니다.

1. html 구조

<html>
    <head> - 화면에 포함되지 않는 메타 데이터 등을 포함하는 태그
       <title> - 브라우저의 탭, 즐겨찾기 등에서 사용되는 태그
       <meta>
       <base> - 문서 안의 모든 상대 URL이 사용할 기준 URL을 지정하는 태그
       <link> - 문서와 외부 리소스의 관계를 명시하는 태그
       <style> - css를 통해 body의 디자인과 레이아웃을 결정하는 태그
    <body> - 실제 브라우저에서 확인 가능한 컨텐츠

html은 텍스트의 구조이자 의미를 나타냅니다.
예를 들어, h1이라는 태그로 묶인 텍스트는 그 자체로 큰 제목이라는 의미이자 동시에 큰 제목이 가져야 할 구조를 나타냅니다.

2. 속성

태그 내부에 선언되어 사용자가 원하는 기준에 맞도록 요소를 설정하거나 다양한 방식으로 동작을 조절하기 위한 값으로,
CSS에서 스타일 적용을 위해 해당 요소를 선택하기 위한 class, id 역시도 속성의 일종입니다.

속성은 프로그래밍 언어의 변수와 같이, 속성명과 그 속성에 대응하는 속성값으로 이루어집니다.
다만 속성명은 이미 정해져있어, 사용자가 임의로 수정할 수 없다는 것이 차이점입니다.

예를 들어, <a> 태그에는 href라는 속성이 존재하며, href 속성의 값으로는 URL이 올 수 있습니다.

※ <a> 태그로 URL을 삽입할 때, URL의 마지막에 왜 '/'를 넣을까?
이때 추가되는 /는 Trailing Slash라고 하며 이것이 붙은 URL은 파일명이 아닌 디렉토리명으로 서버에서 인식하게 됩니다.
즉, Trailing Slash로 서버에서 해당 URL의 종류를 확인하는 과정을 생략하여 통신과정에서 속도 부분의 이점을 얻을 수 있습니다.

속성은 다음과 같은 규칙을 준수하며 작성됩니다.

  1. 속성은 요소 이름과 속성 사이의 공백으로 구분된다.
  2. 하나 이상의 속성이 존재할 경우, 속성의 구분은 공백으로 이루어진다.
  3. 속성의 값은 "따옴표"로 감싸져야한다.


※ vs-code 확장 프로그램

  • open in browser : html 파일을 vs-code에서 alt + B로 바로 실행 가능하게 해주는 확장 프로그램
  • live server : 자신의 컴퓨터를 서버로 하여, html 파일을 실시간으로 반응하는 페이지로 볼 수 있게 해주는 확장 프로그램
  • auto rename tag : 태그 수정 시, 세트로 일치하는 태그도 한 번에 수정해주는 확장 프로그램
  • highlight matching tag : 서로 일치하는 열림태그와 닫힘태그를 표시해주는 확장 프로그램
