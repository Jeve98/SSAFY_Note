RPG + 리듬게임 아이디어 보드

concept : top view + 2d rpg + music game
target : short rpg user

1. 전투에 리듬게임 요소를 추가한다
	[P]
	0. 단조로운 플레이 패턴에 변화
	1. 몬스터별 전용 bgm 사용으로 다채로운 청각적 경험
	2. 동일한 상대라도 키노트의 변화만으로 새로운 전투 경험
	[N]
	0. 매전투 시마다 리듬게임이 추가될 경우, 유의미하지 않은 전투에서 지나친 피로감과 지루함 경험
	1. 잦은 bgm의 변화로 오는 통일감(몰입감) 저하
	2. 다수의 적과 조우 시에 bgm의 중첩

solve_0 : 보스 몬스터에게만 리듬게임을 이식한다.
- N0 : 리듬게임을 유발하는 전투를 극히 일부로 한정하여 피로도를 줄이고 중요한 전투에서 특별한 역할을 제공할 수 있다.
- N1 : 보스 전투의 bgm을 기본으로 리듬게임을 시행하므로 몰입감 저하를 해소할 수 있다.
- N2 : 보스 전투에서만 리듬게임이 진행되므로 다수의 적과 조우했을 경우에도 bgm의 중첩을 해소할 수 있다.
problem : 0) 리듬게임이 장판기와 같은 보스 몬스터의 패턴의 일부가 되는 것이지 게임의 컨셉이라고 볼 수 없어진다.
	      1) 또한 P1과 P3의 긍정적인 경험 역시 저하된다.

solve_1 : 2d rpg가 아닌 전투에 집중시킨 로그라이크 게임으로 개발하며 오픈 월드에서 맵 탐사 기반으로 변경한다.
- N0 : 로그라이크-맵 탐사의 경우, 전투가 한정되므로 피로도가 제한된다.
- N1 : 각 맵의 bgm을 해당 몬스터의 bgm으로 전환하므로 전체적인 몰입감을 유지할 수 있다. (레벨디자인)
- (N2 : 한 맵에서 등장하는 몬스터를 1종, 1마리로 한정할 경우 해소할 수 있다.)
problem : 0) 로그라이크 게임보다도 컨셉이 던전 탐색인 리듬 게임으로 경험될 가능성이 높다.
	      1) N2가 한정적 해소된다. (재미요소~문제점해결 간의 trade-off)
solve_1_0 : 전투적 요소와 로그라이크 요소를 부각하는 것으로 리듬 게임의 경험이 아닌 로그라이크 게임 경험으로 이끌 수 있다.

solve_2 : 특정 스킬에 리듬게임을 이식한다.
- N0 : 해당 스킬을 사용할 때만 리듬게임을 진행하므로 유저의 성향과 상황별 판단에 맞춰 진행할 수 있다.
- N1 : 속칭 '처형곡'과 같은 역할을 하므로 몰입감이 상승한다.
- N2 : 리듬게임이 몬스터에 바인드 된 것이 아니므로 중첩이 발생하지 않는다.
problem : 0) P1을 포기해야 한다.
	      1) 리듬게임이 이식된 스킬을 사용하지 않는 플레이어의 경우, 기존의 게임과 차이를 느끼지 못한다.
	      2) 리듬게임 실패 시의 부정적인 경험이 극대화된다.
solve_2_2 : 리듬게임이 이식된 스킬의 기본 성능을 조절하여 부정적인 경험이 아닌 유저의 실력에 따른 부분으로 변주할 수 있다.
solve_2_2 : 리듬게임 결과에 따라 스킬의 성능을 조절하는 것으로 부정적인 경험을 줄일 수 있다.

solve_3 : 일정 거리에 들어온 몬스터의 bgm을 출력하되, 이미 출력 중인 bgm이 있다면 다른 몬스터의 접근을 일정 거리 이상 막는다.
- N2 : 해소된다.
problem : 0) 선결 과제 N0, N1가 해결되지 않는다.
	      1) '몰려오는 적'이 만들어내는 긴박한 상황이 사라진다. (어떻게 하든 1:1 대치)
solve_3_1 : 접근은 허용하되 출력되는 bgm은 최초 적 조우 시의 bgm만 유지시키고 해당 몬스터 사망 시 다음 몬스터의 bgm을 재생한다.
